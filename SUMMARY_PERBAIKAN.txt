═══════════════════════════════════════════════════════════════════════════════
   🚀 CARA TEST PERBAIKAN
═══════════════════════════════════════════════════════════════════════════════

STEP 1: BUKA FIREBASE CONSOLE
   → https://console.firebase.google.com
   → Pilih project: smartrack-67d7a
   → Klik Firestore Database

STEP 2: BUAT DATA DUMMY
   A. Buat Bus:
      - Klik "Start collection" → Nama: bus
      - Klik "Add document" → Auto-generate ID
      - Tambahkan field sesuai struktur di atas
      - Save & COPY document ID

   B. Buat Perusahaan:
      - Klik "Start collection" → Nama: perusahaan
      - Klik "Add document" → Auto-generate ID
      - Tambahkan field sesuai struktur di atas
      - Save & COPY document ID

   C. Buat Titik Jemput:
      - Klik "Start collection" → Nama: titik_jemput
      - Klik "Add document" → Auto-generate ID
      - Tambahkan field sesuai struktur di atas
      - Di field perusahaan_id: PASTE perusahaan_doc_id dari step B
      - Save & COPY document ID

STEP 3: REGISTER USER BARU (atau gunakan yang sudah ada)
   → Buka aplikasi di Edge
   → Register dengan email & password
   → Login dengan akun tersebut

STEP 4: UPDATE USER DI FIRESTORE
   → Buka collection "users"
   → Cari document dengan email user yang tadi register
   → Klik Edit
   → Tambahkan 3 field:
      • bus_id: [PASTE bus_doc_id dari step 2A]
      • titik_jemput_id: [PASTE titik_jemput_doc_id dari step 2C]
      • perusahaan_id: [PASTE perusahaan_doc_id dari step 2B]
   → Save

STEP 5: HOT RESTART APLIKASI
   → Klik di terminal yang running
   → Tekan 'R' (huruf besar R)
   → Tunggu aplikasi restart

STEP 6: CEK DASHBOARD
   → Login ke aplikasi
   → Dashboard Karyawan sekarang akan menampilkan:
      ✅ Bus Anda: "Bus A-01" (dari Firestore)
      ✅ Titik Jemput: "Gerbang Utama" (dari Firestore)
      ✅ Jam Berangkat: "07:00 WIB" (dari Firestore)
      ✅ Perusahaan: "PT. Industri Jaya" (dari Firestore)

═══════════════════════════════════════════════════════════════════════════════
   💡 TIPS & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

1. Data tidak muncul?
   → Pastikan field bus_id, titik_jemput_id, perusahaan_id sudah ada di user
   → Pastikan Document ID benar (case-sensitive!)
   → Hot restart aplikasi (tekan R)

2. Loading terus?
   → Cek koneksi internet
   → Cek Firebase Console apakah data sudah ada
   → Cek browser console (F12) untuk error

3. Muncul "Error"?
   → Cek Firestore Security Rules
   → Pastikan user sudah login
   → Lihat error detail di browser console (F12)

4. Ingin test dengan cepat?
   → Baca file QUICK_START.md untuk step-by-step guide
